# smart-doc-formatter · 智能文档排版工具

把一份**「内容已写好」的文档** + 一份**目标格式规范**（学校 / 期刊 / 机构）→ 自动套上该规范，输出**「内容一字不丢」的合规成品 + 合规报告**。

核心不是「一个排版工具」，而是 **「规范 → 声明式模板（YAML）」编译器 + 一个可持续增长的模板库**。LLM 只做「结构分类」与「规范抽取」两件判断，其余全是确定性代码——这是「输出即合规、可复现」的根。

## 关键设计（方向，已锁定）

- **建设期 / 运行期两轨**，声明式模板库连接；**workflow 不是 agent**。
- **核心不变量**：内容一个字不动，只换格式层。
- **双校验**：内容审查（确定性 diff，掉内容硬阻断 ✅）+ 格式校验（✅ 出报告；🚧 自动回炉闭环 #6）。
- **LLM 路线 = 接 API**（DeepSeek）；结构分类失败有确定性样式兜底，并**显式标注置信度**（不把降级跑表现成干净跑）。
- 排版 = python-docx + 可程序化生成的 Word 母版 + oxml 补丁（三线表 / 页码 / 文档网格 / 页眉双线）。

## 跑起来

```bash
pip install -r requirements.txt
# 配 .env：DEEPSEEK_API_KEY=...（见 .env.example）
python -m uvicorn app.main:app        # 注意：别带 --reload（编译 / 删除会写文件触发重启）
```

测试（**离线、无需 API key**）：`pytest`

> 平台说明：`.doc → .docx` 转换与页面预览渲染走 PowerShell + Word COM，**需 Windows + 已装 Word**；`.docx` 主链与离线测试跨平台。
